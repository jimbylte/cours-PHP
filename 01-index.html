1 - introduction

Le PHP est un langage de script généraliste et open source. Il est spécialement conçu pour le développement d'application web.

1 - A quoi sert le PHP ?

Comme nous l'avons vu dans l'introduction le php sert a développer des applications web grace a ce langage nous pouvons faire tous type de site, en effet le php est un langage fonctionnel contrairement au html et css.

Grace au php nous pouvons creer tout type de site: moteur de recherche de type google réseaux sociaux comme facebook plateforme multimédia comme youtube ou encore site d'information comme wikipédia. son utilisation ne se limite pas à ça nous pouvons aussi développer des sites e-commerces des espaces membres, des pages d'administration.

Pour mieux comprendre le php va être le langage qui permet aux autres langages de fonctionner. prenons l'exemple d'un formulaire on le style grace au css ou un framework de type bootstrap. Si nous voulons que les données du formulaire soient traitées nous devons utiliser le php qui analysera et traitera les données.

2 - Un site dynamique

Le php nous sert en fait a créer un site dynamique. Un site web dynamique va etre un site dont les informations proviennent d'une base de données (BDD) et pour ce la on va apprendre a faire fonctionner le php et les sql main dans la main.

Attention, il ne faut pas confondre un site web dynamique et un site web d'animation géré par des écouteurs d'évènements comme peut le faite le javascript.

Nous utiliserons à partir de maintenant l'appellation site dynamique pour les sites dont les informations sont stockées en BDD et qui possèdent deux interfaces distinctes :

  1. Une interface "FRONT" sur laquelle on retrouvera l'affichage "normal" du site le coté pour les admin et les clients => on l'appelera front-office
  2. Une interface côté back qui permettra d'assurer la gestion de la base de données et certains réglages de l'interface front=> on l'appelera back-office

3 - Notions importantes

Vous ne verrez jamais du code PHP dans le code source d'une page web car ce dernier est un language interprété par le navigateur et ne renvoie (génère) que du code HTML. C'est la raison pour laquelle les fichiers ne sont plus lancés comme avant à partir de l'explorateur de fichiers, et qu'ils doivent tous être placés dans le dossier HTdocs de XAMPP ou WWW dans l'aragon par ex. Pour lancer les sites, il nous faudra un serveur local et on aura deux manières de les ouvrir:

  • Depuis XAMPP -> HTdocs puis le dossier concerné
  • Depuis le navigateur -> URL:localhost/nom du dossier

Un site dynamique n'utilise pas qu'un seul language, il faut marier le HTML, le CSS, le PHP, le SQL, le JS, etc.

4 - Serveur et difference entre PHP procédural et PHP orienté objet (POO)

Un serveur est un ordinateur qui stocke un site mis à disposition par un hébergeur. Cet ordinateur reste tjrs allumé pour que les sites mis à disposition soient consultables 24/24 et 7/7. Un serveur local va reproduire les fonctions d'un serveur à petite échelle sur notre ordinateur (MAMP, WAMP, XAMPP, laragon; ...).

Il existe deux techniques de développement en PHP :

  • L'approche procédurale dans laquelle le code sera écrit de façon séquentielle.
  • L'approche orientée objet (POO) : Le code est alors encapsulé dans des méthodes, des classes et fonctionne grâce à l'interraction d'objets.

    • Certaines méthodes de bases doivent forcément être aprises grâce au procédural. L'orienté objet ne sert pas à "aller pluss loin", ce sont juste deux méthodologies differentes mais qui permettent de faire la même chose.

    Certaines grandes entreprises préfèrent tout de même travailler avec l'orienté objet car cette méthode encourage le travail de groupe et facilite les mises à jour sur un travail de longue durée.

    Ces deux méthodes peuvent être utilisées pour développer un site "from scratch" (de 0) dans un CMS ou encore dans un framework quadriciel (symfony utilise naturellement l'approche orientée objet cependant on peut faire du procédural avec).
